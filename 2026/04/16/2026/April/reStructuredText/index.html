reStructuredText

發表於 2026-04-16 分類於 Software Engineering ， Web Development ， Sphinx 文章字數： 727 所需閱讀時間 ≈ 1 分鐘

.rst 檔案是 reStructuredText 的縮寫。它是一種輕量級標記語言（Lightweight markup language）的純文字檔案，概念上與 Markdown（.md）非常類似。

1. 主要用途

  • Python 社群的標準文件格式： 它是 Python 程式語言界最常使用的文件編寫格式。Python 的官方文件，以及絕大多數 Python 開源專案（如 Django、Flask 等）的說明文件，都是使用 .rst 格式撰寫的。
  • 搭配 Sphinx 工具： 開發者通常會寫好一堆 .rst 檔案，然後使用一個叫做 Sphinx 的工具，將這些純文字檔自動轉換成漂亮的 HTML 網頁、PDF 或電子書格式（你在 Read the Docs 網站上看到的文件大多是這樣來的）。
  • README 說明檔： 在 GitHub 等平台上，除了常見的 README.md，也經常會看到 README.rst 作為專案的介紹首頁。

2. 檔案特色

  • 純文字格式： 內容完全由純文字組成，非常容易被版本控制系統（如 Git）管理。
  • 高可讀性： 即使沒有經過轉換，直接閱讀原始的 .rst 原始碼也能輕易看懂內容結構。
  • 擴充性強： 相比於 Markdown，reStructuredText 提供了更嚴謹且強大的語法，特別適合用來編寫包含大量交互連結、目錄、複雜表格、索引的「大型技術手冊」。

3. 如何開啟與編輯？

因為它是純文字檔，你可以用任何文字編輯器來開啟它，例如：

  • Windows 內建的「記事本」或 Mac 的「文字編輯」
  • Visual Studio Code (VS Code) （強烈推薦，可安裝 reStructuredText 擴充套件來提供語法亮點和即時預覽功能）
  • Sublime Text
  • Notepad++
