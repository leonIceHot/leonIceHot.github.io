Kubernetes

發表於 2023-07-13 分類於 Network 文章字數： 3.4k 所需閱讀時間 ≈ 6 分鐘

簡介

Kubernetes（中文譯為「容器集群管理系統」）是一個開源的容器管理平台，用於自動化部署、擴展和管理容器化應用程序。它提供了一個可靠的容器環境，能夠跨多個主機進行應用程序的部署和運行。

功能

Kubernetes 的設計目的是簡化容器應用程序的部署和管理。它提供了一個強大的平台，使開發人員和運維團隊能夠輕鬆地管理數百個甚至數千個容器化應用程序的部署。Kubernetes 使用一個稱為「Pod」的最小部署單位，該單位可以包含一個或多個容器。這些 Pod 可以在集群中的不同主機上運行，並且可以根據需求自動擴展或縮放。

Kubernetes 還提供了一個豐富的功能集，包括容器自動補充、容器健康檢查、負載平衡、服務發現、配置管理和存儲管理等。它具有高度可擴展性和彈性，能夠應對大規模應用程序的需求。

使用 Kubernetes 可以獲得許多好處。它可以提供高可用性和容錯能力，使應用程序能夠持續運行，即使某些主機或容器發生故障。同時，Kubernetes 還提供了彈性和可伸縮性，可以根據流量和資源需求自動調整應用程序的規模。

總結來說，Kubernetes 是一個開源的容器集群管理平台，用於自動化部署、擴展和管理容器化應用程序。它提供了豐富的功能集，使開發人員和運維團隊能夠輕鬆地管理大規模容器化應用程序，同時獲得高可用性、彈性和可伸縮性。

安裝

Ubuntu 22.04
Kubernetes 1.27

Step 1 - Update Ubuntu
sudo apt update
sudo apt upgrade

Step 2 - Install Docker

Set up the repository

Update the apt package index and install packages to allow apt to use a repository over HTTPS:

sudo apt-get update
sudo apt-get install ca-certificates curl gnupg

Add Docker’s official GPG key:

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg –dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

Use the following command to set up the repository:

echo
“deb [arch=”$(dpkg –print-architecture)” signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu
“$(. /etc/os-release && echo “$VERSION_CODENAME”)” stable” |
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

Install Docker Engine

Update the apt package index:

sudo apt-get update

Install Docker Engine, containerd, and Docker Compose.

sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

Verify that the Docker Engine installation

sudo docker run hello-world

https://docs.docker.com/engine/install/ubuntu/

Step 3 - Start and Enable Docker
sudo systemctl enable docker
sudo systemctl start docker
sudo systemctl status docker

Step 4 - Install Kubernetes

Download the latest release

curl -LO “https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

Validate the binary (optional)

curl -LO “https://dl.k8s.io/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl.sha256"
echo “$(cat kubectl.sha256) kubectl” | sha256sum –check

Install kubectl

sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

Test to ensure the version

kubectl version –client

https://kubernetes.io/docs/tasks/tools/install-kubectl-linux/

Step 5 - Kubernetes Installation Tools (kubeadm、kubelet and kubectl)
kubeadm: the command to bootstrap the cluster.
kubelet: the component that runs on all of the machines in your cluster and does things like starting pods and containers.
kubectl: the command line util to talk to your cluster.

Update the apt package index and install packages needed to use the Kubernetes apt repository

sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl

Download the Google Cloud public signing key:

curl -fsSL https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo gpg –dearmor -o /etc/apt/keyrings/kubernetes-archive-keyring.gpg

Add the Kubernetes apt repository:

echo “deb [signed-by=/etc/apt/keyrings/kubernetes-archive-keyring.gpg] https://apt.kubernetes.io/ kubernetes-xenial main” | sudo tee /etc/apt/sources.list.d/kubernetes.list

Update apt package index, install kubelet, kubeadm and kubectl, and pin their version:

sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

https://kubernetes.io/docs/setup/production-environment/tools/kubeadm/install-kubeadm/
